1. INTRODUCTION

xOS is built on HTML, PHP, JavaScript, and Ajax. You may incorporate any of these elements. Up to what version may you

use? Our server stays up to date with most new web technologies including: PHP 5.2, Perl 5.8, and Python 2.4.
We offer hosted apps for apps smaller than 1MB(including images). You may also host your own app on your website and

submit your link via email to app submit@xproduct.freehostingcloud.com. (More on this subject is discussed in Submitting

Your App section.)

2. Getting Started

---------------------------

We are going to make a pretend application throughout this guide.

NAME= Horseback Guide

---------------------------

We suggest that you design your app as a normal webpage and then scale it to fit in a window in xOS. The reason behind

this is that windows are resizable and we are also planning to integrate fullscreen apps in the near future. You may write

your application with one or a combination of any of the following programing languages:
HTML
PHP
Ajax
Perl
Phython
JavaScript

---------------------------

a.Code Requirements

Design you app any way you want. Requirements:

1) If you will have elements that do more then what they should usually do (example: a div that drags across the screen;

a span that displays a script) please put the first three letters or any three constants from you applications name.

---------------------------

From our development stage of Horse back Guide we have this div that is going to drag across the app.

<div id="drag" class="draggable-div"...

NOTE: The generic id of the above div. Instead of drag we could take some main letters from the title: Horse Back Guide.

<div id="hbgdrag" class="draggable-div"...

---------------------------

2) If you have elements that show, hide, or toggle visibility do the same as stated above.

---------------------------

From our development stage of Horse back Guide we have this div that is going to drag across the app.

<div id="show" style="visibility:hidden"...

NOTE: The generic id of the div. Instead of drag we could take some main letters from the title: Horse Back Guide.

<div id="hbgshow" style="visibility:hidden"...

---------------------------

3) Do not develop EXTERNAL CSS that changes the application window UI. WE WILL NOT ATTACH THEM. You may however change the

CSS directly in the application style attribute.

4) You may develop EXTERNAL SCRIPTS that are specific to your application. WE WILL NOT USE SCRIPTS THAT INTERFER WITH OUR

SCRIPTS. You may not use at this time the Ajax Google API for JQuery any version. We already use 1.4.2 and 1.5.1

jquery.min.js.

---------------------------

3. Window Format

[windowname] = Your applications ALL lowercase letters name ONLY. MUST NOT exceed 20 Characters. NO ADDED HTML TAGS.

PLAIN TEXT ONLY
[fullwindowname] = Your applications full name may include uppercase, lowercase, and numbers. MUST NOT exceed 76

Characters. NO ADDED HTML TAGS. PLAIN TEXT ONLY.
[windowcontent] = The content is almost anything you can think of using HTML, PHP, JavaScript. Perl, Phython, Ajax. No

Character Limit. You may add anything you would like as long as it is within the Code Requirements.

The window format:

<div id="[windowname]" style="overflow:auto" class="window">
<div class="menubar"
onmousedown="dragStart(event, '[windowname]')">
<a href="JavaScript:hideDiv('[windowname]')"><img src="CORE/Close.png" width="16" height="16" align="right" alt="Close"
onmouseover="this.src='CORE/CloseBack.png'"
onmouseout="this.src='CORE/Close.png'"></a> [fullwindowname]</div>
<div class="content">
<p>[windowcontent]</p>
</div>
</div>

:END WINDOW FORMAT

[WINDOWNAME]*
[WINDOWCONTENT]#
[FULLWINDOWNAME]$

* 3 total of this field
# 1 total of this field
$ 1 total of this field

Make sure all fields in this div are filled out properly.

4. API's

---------------------------

We are going to discuss, in brief, some different API's specific to xOS Mobile.

a. Visible Elements API

In xOS Webtop or Mobile we use a JavaScript that toggles any elements CSS Property of visiblity.

Most of the applications onLoad have the CSS Property of visibility:hidden; This is how you see the home screen

onUnlock. The only element that doesn't have this property is, yes you guessed it, the lock screen.

We hide or show certain elements by either a link or a onClick JavaScript call.

---------------------------

Ex. 1 - HREF

to hide:
<a href="JavaScript:hideDiv('id-of-element')">Image or Word(s)</a>

to show:
<a href="JavaScript:showDiv('id-of-element')">Image or Word(s)</a>

Ex. 2 - onClick

to hide:

<a onClick="hideDiv('id-of-element')">Image or Word(s)</a>

to show:

<a onClick="hideDiv('id-of-element')">Image or Word(s)</a>

---------------------------

I am sure you know that you can onClick anything really.

That is our Visible Elements API

b. IScroll 4 API

We can't explain it much better than he can. So look at cubiq.org/iscroll-4 for further documentation. Great documentation

by the way.

We do however have something special. Carousels are optimized to fit 320px across. The full width of the iPhone Screen.

c. jQuery

We use jQuery 1.5.1. Nothing fancy here. Best documentation at jquery.com.

d. Background Change API

When you click on the link in settings to change the background it is really just a simple JavaScript call:

 

Change with HREF

<a href="JavaScript:changeBGImage(0);">LINK TEXT</a>

Change with onClick

<a onClick="changeBGImage(0);">LINK TEXT</a>

The only thing is that images that are in the background change group are globally defined in JavaScript So in order for your to get a image to be in the background you must submit the url of the image to us so we may add it to our database as a background image. We will explain how to do this in the App Submision Part.

2. Getting Started

---------------------------

We are going to make a application throughout this guide.

NAME= Play Hockey

---------------------------

We suggest that you design your app as a normal mobile web page and then scale it to fit in a application window later in

xOS. The reason behind this is that you will find more ease in this and you will not stress yourself out over the

smallness. We will get to smallness in a second. You may write your application with one or a combination of any of the

following programing languages:

HTML
PHP
Ajax
Perl
Python
*avaScript

---------------------------

a.Code Requirements

Design your Application any way you would like but in-order for your app to be successful these are the

requirements:

1) You are REQUIRED to use the following code when going to links of other pages in you app:

JavaScript:window.location.href = 'http://url.to/your.page';

This is extremely important and we cannot stress it enough. If you don't use the above code your app and the Webtop OS

will break and go out of xOS Mobile.

2) USE OFFLINE MANIFEST CACHE!!!! If you don't you will crash the Webtop OS. If you can't or are unable to use Offline

Manifest Cache, please make a launch page with a warning saying:

If you are offline, please connect to the internet to use this app.

3) If you are developing in an iFrame please add: <meta id="viewport" name="viewport" content ="width=device-width,

minimum-scale=1.0, maximum-scale=1.0, user-scalable=no"/> to your head tag.

4) That's just about it. If your application is in an iframe your all good and have nothing to worry about if you follow

the first two rules.

way the Webtop loads, looks, or feels, without first making the user aware.

|LINK 4 to change BG|-TAP
-Are you sure?
Y or N

------------------------------
END APP
-------

You must give the user the option before changeing any system prefences that are not defined in the xOS Settings. Oh and

yes their is an API for changing backgrounds of the system. We will discuss API's later.

a. Exceptions

You may change the way that the Back Button Functions with the following restrictions:
1) On any page of your application the back button must take them back to the LAST page that was viewed.
2) If you have a form that is multiple pages the back button MAY act as a cancel button.

NOTE: You may NEVER change the function of the HOME button.

b. IScroll 4

IScroll is a wonderful tool. We are very fortunate to have Matt eo Spinelli keep a log of the wonderful things he does at cubiq.org. Without the use of iScroll 4 xOS Mobile would not be were it is today. We highly suggest you use IScroll. You may use any version you want in iFrames but for those how want to build in we have API's for you.

c. jQuery

jQuery is also a wonderful tool. Again if you are iFrame do everything yourself. If you are building in use jQuery 1.5.1 API.

Application Submission - All Products - Global

---------------------------

Once you have designed your application and tested it.(You can test your applications by going to your application web address in xOS Mobile or by plugging in the code for your application using Google Chromes Web Console.) You can submit your application to bdavis@xproduct.freehostingcloud.com.

Provide the subject: NEW Application: NAME OF APPLICATION
Provide the content:

For Iframes; URL: http://url.of/application, Designer Name/Company, Mobile or Desktop

For Build-ins; CODE: xOS Optimized Code, Designer Name/Company, Mobile or Desktop, What API's you will be using, Dont forget to include a manifest file. All files must be included in a .zip or .tar.gz file type. All others will be refused.

When you app is read over, you will most likely receive a Accepted email or a improve function of certain element it currently conflicts with the OS. On rare occasion will you receive a rejection email.
